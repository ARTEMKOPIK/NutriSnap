# Увеличение памяти для сборки (Исправление ошибки JVM)

Ошибка `JVM garbage collector is thrashing` означает, что процессу сборки Gradle не хватает оперативной памяти. На GitHub Actions по умолчанию выделяется слишком мало памяти (512 МБ), чего недостаточно для современного Android-проекта.

## Мой план решения:

1. **Настройка [gradle.properties](file:///c:/Users/artemkopik/Desktop/Новая%20папка/gradle.properties)**:
   - Я добавлю параметр `org.gradle.jvmargs=-Xmx2048m -XX:MaxMetaspaceSize=512m`.
   - Это увеличит доступную память для сборки до **2 ГБ**, что является стандартом для стабильной работы.

2. **Синхронизация**:
   - Отправлю изменения в репозиторий. GitHub Actions перезапустит сборку с новыми настройками памяти, и она должна пройти успешно.

## Результат:
Сборка перестанет падать из-за нехватки памяти, и ты получишь свой APK.
