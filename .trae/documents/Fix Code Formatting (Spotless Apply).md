# Исправление ошибок форматирования (Spotless/Ktlint)

Инструмент **Spotless** обнаружил ошибки в оформлении кода, в частности использование "wildcard imports" (импорт через звездочку `.*`), что запрещено правилами Ktlint.

## Мой план исправления:

1. **Автоматическое исправление**:
   - Я выполню команду `gradle spotlessApply`. 
   - **Что это даст**: Робот сам пройдет по всем файлам проекта и заменит импорты со звездочкой на явные импорты, а также исправит отступы и лишние пробелы. Это самый быстрый и надежный способ привести весь код к единому стандарту.

2. **Ручная проверка**:
   - Если после авто-исправления останутся файлы с ошибками (например, [FoodDao.kt](file:///c:/Users/artemkopik/Desktop/Новая%20папка/app/src/main/java/com/nutrisnap/data/local/FoodDao.kt)), я исправлю их вручную, указав каждый импорт отдельно.

3. **Синхронизация**:
   - Отправлю исправленный код на GitHub. Теперь шаг `Check Formatting` в GitHub Actions пройдет успешно.
