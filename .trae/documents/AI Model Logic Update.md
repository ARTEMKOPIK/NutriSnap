# Обновление логики выбора моделей ИИ

Для оптимизации работы приложения под разные типы запросов, я внесу следующие изменения:

## Изменения в коде:
1. **Динамический выбор модели в [GroqRepository.kt](file:///c:/Users/artemkopik/Desktop/Новая%20папка/app/src/main/java/com/nutrisnap/data/repository/GroqRepository.kt)**:
   - Если пользователь отправляет **фото**, будет использоваться мощная мультимодальная модель `meta-llama/llama-4-maverick-17b-128e-instruct`.
   - Если пользователь отправляет **только текст**, будет использоваться модель `llama-3.3-70b-versatile`, которая лучше справляется с текстовыми описаниями.

2. **Обновление структуры запроса**:
   - Настройка [GroqRequest](file:///c:/Users/artemkopik/Desktop/Новая%20папка/app/src/main/java/com/nutrisnap/data/api/GroqModels.kt) для корректной передачи выбранной модели в API.

## Техническое обоснование:
- `llama-3.3-70b-versatile` — одна из лучших моделей для понимания текста, что идеально подходит для анализа текстовых дневников питания.
- `llama-4-maverick-17b-128e-instruct` — новейшая мультимодальная модель, обеспечивающая высокую точность распознавания объектов на фото.
