# Настройка Автоматизации (CI/CD) через GitHub Actions

Чтобы GitHub сам проверял код и собирал приложение при каждом обновлении, я создам специальный файл конфигурации **GitHub Actions**.

## Что будет сделано:

1. **Создание [.gitignore](file:///c:/Users/artemkopik/Desktop/Новая%20папка/.gitignore)**:
   - Добавлю файл, который скажет Git игнорировать временные файлы Android Studio и Gradle. Это нужно, чтобы в репозиторий попадал только чистый код.

2. **Создание рабочего процесса (Workflow)**:
   - Создам файл `.github/workflows/android_release.yml`.
   - **Как это будет работать**:
     - **Триггер**: Срабатывает каждый раз, когда ты делаешь `git push` в основную ветку (`main`).
     - **Проверка**: GitHub запустит автоматическую проверку кода на ошибки (Lint) и сборку проекта.
     - **Сборка**: Если ошибок нет, GitHub соберет готовый **APK-файл**.
     - **Релиз**: GitHub создаст новый «Релиз» (Release) в твоем репозитории и прикрепит туда этот APK.

3. **Инструкция по секретам**:
   - Поскольку в коде есть API-ключ Groq, я объясню тебе, как в будущем перенести его в «Secrets» на GitHub, чтобы он не лежал в открытом виде (если планируешь сделать репозиторий публичным).

## Что тебе нужно будет сделать:
После того как я создам файлы, тебе нужно будет:
1. Создать пустой репозиторий на GitHub.
2. Инициализировать Git в этой папке: `git init`.
3. Привязать репозиторий: `git remote add origin <твоя_ссылка>`.
4. Отправить файлы: `git add .`, `git commit -m "Initial commit"`, `git push origin main`.
