# Повышение автономности проекта (Максимальная автоматизация)

Чтобы проект жил и обновлялся практически без твоего участия, я внедрю следующие «умные» функции автоматизации:

## Что я сделаю прямо сейчас:

1. **Авто-обновление библиотек (Dependabot)**:
   - Создам файл `.github/dependabot.yml`.
   - **Как это работает**: Робот GitHub будет раз в неделю проверять, вышли ли новые версии Android-библиотек (Retrofit, Room, Compose). Если вышли — он сам создаст запрос на обновление, тебе останется только нажать «Принять».

2. **Автоматическое ведение списка изменений (Changelog)**:
   - Обновлю [android_release.yml](file:///c:/Users/artemkopik/.github/workflows/android_release.yml).
   - **Как это работает**: Теперь в описании каждого релиза на GitHub будут автоматически перечислены твои сообщения из коммитов. Тебе не нужно писать описание релиза вручную.

3. **Авто-нумерация версий**:
   - Обновлю [build.gradle.kts](file:///c:/Users/artemkopik/app/build.gradle.kts).
   - **Как это работает**: Версия приложения (Version Code) будет увеличиваться сама при каждой сборке на GitHub. Тебе не нужно менять цифры в коде.

4. **Авто-проверка качества кода (Lint)**:
   - Добавлю шаг проверки в GitHub Actions.
   - **Как это работает**: Перед сборкой GitHub проверит код на ошибки и потенциальные баги. Если код «грязный», сборка остановится и пришлет тебе уведомление.
