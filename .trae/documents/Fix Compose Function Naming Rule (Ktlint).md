# Исправление правил именования функций для Compose (Spotless/Ktlint)

Инструмент **Spotless** выдает ошибку, так как по умолчанию он ожидает, что все функции начинаются с маленькой буквы. Однако в **Jetpack Compose** функции-компоненты (как `CameraScreen`) принято называть с большой буквы.

## Мой план исправления:

1. **Создание конфигурации [.editorconfig](file:///c:/Users/artemkopik/Desktop/Новая%20папка/.editorconfig)**:
   - Создам файл в корне проекта с правилом `kotlin_function_naming_allow_pascal_case = true`. 
   - Это скажет линтеру, что именование функций с большой буквы разрешено, что является стандартом для Compose.

2. **Обновление настроек Spotless**:
   - Убежусь, что Spotless учитывает эти настройки.

3. **Синхронизация**:
   - Отправлю исправление на GitHub. После этого проверка `spotlessCheck` пройдет успешно, так как мы легализовали стиль именования Compose.
